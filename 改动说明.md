### 1) environment.yml
- 改动 ：把 pyqt>=5.15 改成 pyqt=5.15.* 。
- 原因/效果 ：强制使用 Qt5/PyQt5 对应版本，避免 conda 自动装 PyQt6 导致 ModuleNotFoundError: PyQt5.QtWidgets 。
### 2) utils/models.py
- 改动 ：扩展设备/进路模型字段与枚举。
- 新增/扩展点 ：
  - SignalAspect （STOP/PROCEED/CAUTION/DOUBLE_CAUTION/SHUNT_PROCEED/GUIDE）
  - RouteStage （IDLE/SELECTED/PRELOCKED/OPENED/APPROACH_LOCKED/RELEASING/RELEASED）
  - 信号：封锁/断丝/故障字段；增加 MOON_WHITE 色。
  - 区段：分路不良/故障字段。
  - 进路：接近/离去区段、敌对进路、起始显示、人解倒计时字段。
- 原因/效果 ：为“完整联锁流程+仿真故障/封锁”提供状态承载，避免业务逻辑散落在 UI/日志字符串里。
### 3) utils/station_config.json
- 改动 ：新增站场静态配置（信号/道岔/区段/进路关系）。
- 内容 ：每条进路包含 path_tracks / approach_tracks / departure_tracks / switches / flank_switches / conflicts / start_aspect。
- 原因/效果 ：把“进路表”从逻辑层硬编码迁移到配置驱动，后续对齐老师站场图主要改这个文件而不是改代码。
### 4) utils/state_manager.py
- 改动 1 ： init_station_data() 改为从 station_config.json 加载设备与进路（替换原硬编码初始化）。
- 改动 2 ：新增 update_route() 、 find_route() 、 list_routes_from_start() 。
- 改动 3 ： update_signal/update_switch/update_track/update_route 增加“值变化才 emit state_changed”的逻辑。
- 原因/效果 ：
  - 解决“联锁订阅 state_changed 后因为自己 update 又触发回调造成递归/重复日志”的问题。
  - 支撑 UI 在点选始端时提示“可选终端”。
### 5) simulation/device_simulator.py
- 改动 ：
  - 道岔转换延时改为 3~5s（随机），并保留 MOVING 状态。
  - 道岔完成/失败的 日志输出顺序 调整：先 log“转换完毕/失败”，再 update_switch ，避免日志顺序与联锁推进交错造成误解。
  - 新增 set_signal_aspect() ：以显示语义驱动颜色（列车红/绿/黄、调车蓝/白、引导月白）。
  - set_track_occupancy() 支持可选延时。
  - start_manual_unlock_countdown() ：人解倒计时由仿真层 QTimer 托管，每秒写回 route.remaining。
- 原因/效果 ：把“延时/物理动作”集中在仿真层，联锁层只下发动作；同时保证日志能反映真实时序。
### 6) logic/interlocking_engine.py
- 大改动：从硬编码 route_table → 配置驱动 routes
  - try_set_route() 从 StateManager.find_route() 获取 RouteModel，不再使用旧的 route_table 。
- 新增/完善的核心行为
  - 前置校验：区段占用/锁闭/封锁/故障/分路不良，道岔锁闭/单锁/封锁/故障/转换中，信号断丝等。
  - 敌对进路拦截：conflicts 中任一激活则拒绝办理。
  - 选排→转岔→预锁→开放：道岔到位后进入预锁并开放信号。
  - 压入关闭：列车压入进路首区段后自动关闭始端信号。
  - 分段解锁（三点检查）+ 末段出清后整条进路解锁。
  - 取消进路：仅 SELECTED/PRELOCKED 且全空闲时允许。
  - 总人解：按类型触发 180s/30s 倒计时，倒计时结束且区段空闲后解锁。
  - 区故解（单区段解锁）入口已具备。
- 修复/增强
  - 修复“预锁/开放重复打印”：通过在锁闭前先切换 RouteStage 来阻断重入。
  - 增强日志：输出“道岔条件：…已到位/待转换/防护”、输出“锁闭设备：区段[…] 道岔[…]”。
  - 锁闭区段列表去重（避免日志出现 JXG 重复）。
- 原因/效果 ：联锁主链路可演示、且日志能解释“为什么这样做”，便于验收。
### 7) ui/main_window.py
- 改动 ：
  - UI 选路：第一次点始端后在提示栏显示“可选终端”，避免用户乱点导致一堆“未定义进路”。
  - 办理失败时不清空始端，方便重新点终端。
  - 将 UI 的单操/总取消/总人解/引导/引导总锁/模拟占用统一收敛到 InterlockingEngine 接口。
  - 新增“清日志”功能，并修复了启动崩溃问题： log_view 创建前不能直接 connect self.log_view.clear ，改为连接到安全方法 _clear_log() 。
- 原因/效果 ：UI 更像教练机操作习惯（先给可选项），并保证 UI 不直接改状态。
