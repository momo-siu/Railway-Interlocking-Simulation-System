# 计算机联锁模拟仿真系统 (Railway Interlocking Simulation System)

本项目是一个纯软件实现的铁路计算机联锁模拟仿真系统，旨在模拟标准铁路中间站的行车联锁逻辑、设备状态及上位机操作。适用于轨道交通专业教学演示与课程作业。

## 🚉 站场布局
本站为标准中间站，包含以下设备：
- **股道区段**: 3条到发线 (IIG正线, 1G, 3G) + 1条安全线 + 进站内方/接近区段 (JXG, IIAG, IIBG, JSG)。
- **信号机**: 进站信号 (X/S), 出站信号 (X3/XII/X1, S3/SII/S1), 调车信号 (D1/D2)。
- **道岔**: 1号、2号、3号、4号、5号道岔。

## ✨ 功能特性
### 1. 上位机图形化界面
- **实时显示**: 采用不同颜色区分设备状态（灰色-空闲，红色-占用，白色-锁闭，黄色-转换中）。
- **交互操作**: 支持点击信号机办理进路，支持道岔单操、总取消等功能。
- **操作日志**: 实时记录系统动作、联锁校验结果及错误报警。

### 2. 后台联锁逻辑
- **进路办理**: 自动进行联锁检查（区段空闲、道岔位置、敌对进路）。
- **设备锁闭**: 进路建立后自动锁闭相关道岔与轨道区段，开放信号机。
- **解锁逻辑**: 支持总取消人工解锁。

### 3. 现场设备仿真
- **道岔仿真**: 模拟道岔转动的物理延时效果（约2秒）。
- **信号仿真**: 模拟信号机颜色的动态切换。
- **轨道仿真**: 支持手动模拟轨道区段的占用与出清。

## 🏗️ 项目架构
系统采用分层解耦架构设计：
- `ui/`: 界面层，基于 PyQt5 实现。
- `logic/`: 逻辑层，负责联锁运算引擎。
- `simulation/`: 仿真层，模拟现场物理设备动作。
- `utils/`: 通用工具，包含状态管理器 (StateManager) 与数据模型。

## 🚀 环境配置与运行

### 方法一：使用 Conda (推荐)
```bash
# 创建并激活环境
conda env create -f environment.yml
conda activate rail

# 启动系统
python main.py
```

### 方法二：使用 Pip
```bash
# 安装依赖
pip install -r requirements.txt

# 启动系统
python main.py
```

## 🎮 使用指南
1. **办理接发车进路**:
   - 点击始端信号机（如 `X`），提示栏显示已选择始端。
   - 点击终端信号机（如 `XII`），系统开始自动检查并转换道岔。
   - 观察道岔变黄（转换中）后变白（锁闭），最后信号机变绿。
2. **道岔单操**: 使用界面右侧的控制面板，可以对 1-5 号道岔进行定操或反操。
3. **模拟占用**: 点击界面右下角的“占用/出清”按钮，可以手动模拟列车进入或离开接近区段。
4. **总取消**: 点击“总取消”按钮可清除当前所有进路状态并关闭信号。
